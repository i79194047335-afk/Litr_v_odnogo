# lighter-rangebar-scalper

Исследовательский проект: бэктест range-bar скальпинг-стратегии (семейство Lance Beggs / YTC Scalper) под перпы Lighter.

> **Scope v0 — намеренно без живой торговли.** Здесь только сбор данных и бэктест. Кода, который шлёт реальные ордера, в репозитории нет. Это не упущение, а правильный порядок сборки: сначала данные → бэктест → пейпер, и только потом разговор про исполнение. Это не финансовая рекомендация.

## Зачем VPS уже сейчас
Не ради торговли. У Lighter нет минутных свечей под range bars — их строят из **потока сделок**. Чтобы накопить историю тиков, нужна машина, работающая 24/7. Поэтому первое, что крутится на VPS, — это **сборщик тиков**, а не торговый бот.

## Структура
```
src/
  collector/   lighter_ticks.py   — WS-сборщик сделок с Lighter (СТАБ: нужен реальный SDK)
  rangebars/   builder.py         — построение range-баров из тиков (готовая логика)
  indicators/  (EMA, Keltner, Stochastic)
  backtest/    (событийный бэктест WF-ядра + модель комиссий/слиппеджа/fill-rate)
data/          сырые тики / собранные бары (в .gitignore)
scripts/       run_collector.sh и пр.
tests/         юнит-тесты индикаторов и баров
config.example.yaml / .env.example
```

## Установка
```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env            # вписать ключи (НЕ коммитить)
cp config.example.yaml config.yaml
```

## Запуск сборщика на VPS
```bash
# фоном с логом
nohup python -m src.collector.lighter_ticks > /var/log/lighter-ticks.log 2>&1 &
tail -f /var/log/lighter-ticks.log
# либо как systemd-сервис — см. scripts/lighter-ticks.service
```

## Создать репозиторий на GitHub и запушить
Удалённый репозиторий придётся создать **под своим аккаунтом** — у меня нет доступа к твоему GitHub.
```bash

```
(Если есть `gh` CLI: `gh repo create lighter-rangebar-scalper --private --source=. --push`.)

## Workflow (git — источник правды, VPS — раннер)
1. Репозиторий создаётся на GitHub. На VPS: `git clone` / `git pull`.
2. Файлы и команды создаются на VPS через bash → `git add` → `git commit` → `git push`.
3. VPS запускает только то, что уже в репо (сборщик тиков как systemd-сервис — см. `scripts/lighter-ticks.service`).
4. Разработка бэктестера живой торговли не касается — гоняется на собранных данных где угодно.
5. AI-агент (Codex/Claude Code) используется как инструмент, который **коммитит в git и даёт прочитать дифф**, а не правит прод и сам запускает бота.

## Защита от утечки секретов
`.gitignore` — НЕ защита (не остановит ключ, захардкоженный в `.py`). Реальная защита:
- **Локально:** `pre-commit` + gitleaks (см. `.pre-commit-config.yaml`). `pip install pre-commit && pre-commit install`. Сканирует дифф до коммита, бесплатно, на любом репо.
- **Сервер:** GitHub Action gitleaks (`.github/workflows/gitleaks.yml`) как бэкстоп.
- Родная push-protection GitHub бесплатна только для **публичных** репо; для приватного — платный Secret Protection. Поэтому опираемся на gitleaks.
- Если секрет попал в историю — **ротация ключа обязательна**, чистки истории недостаточно.

## Заметки по Lighter (из ресёрча — учитывать при разработке)

Технические факты по площадке:
- **Сеть:** технически Arbitrum. Депозит нативным ETH бриджится в Arbitrum; торгуется **USDC**.
- **ZK-движок → почти анонимно:** публичного трекинга индивидуальных сделок нет. Следствие для кода: **чужие сделки per-account собрать нельзя**; для своих ренж-баров используем **публичный поток сделок рынка** (он есть). Per-account слежки/копи «вручную» не построить.
- **Копитрейд:** официальный инструмент — **Public Pools** (`app.lighter.xyz/public-pools`). Если когда-нибудь понадобится копирование — это штатный путь, а не парсинг чужих сделок.
- **Чего на Lighter НЕТ:** встроенных ренж-баров, 15-сек ТФ, MT5. Ренж-бары строим сами из тиков (наш `rangebars/builder.py`). Для ручной сверки графика — `gocharting.com` (free, есть ренж-бары).
- На графике Lighter не отображаются ask/bid и точки/время входа — для анализа исполнения нужен свой лог.
- Встроенный риск-калькулятор: вводишь стоп → показывает риск в $/%, ползунком объём. Изолированная маржа, плечо на макс.

## Грабли (подсмотрено у тех, кто уже гонял бота на Lighter)
- **Главная:** на коротких интервалах бот показывает профит, а на **суточной дистанции разваливается** — влезают тормоза и подвисания. → нагрузочное/суточное тестирование обязательно ещё до реала.
- **VPS:** нужен забугорный **hi-CPU** (скальперу важен процессор), оплата из РФ проходит у Aeza (~200₽ промо, Стокгольм), firstvds (NL/KZ). VPN к Lighter иногда роняет подключение — закладывать авто-reconnect.
- **Трал позиции важнее точности входа:** у практика доработка трейлинга дала ×5 к суточному PnL. В нашем бэктесте трал/сопровождение моделировать так же тщательно, как вход.
- **Анти-паттерн, который НЕ повторяем:** там бот целиком сгенерён ИИ (Codex/ChatGPT/Claude Code) без понимания кода, авто-запуск после правок, сразу на реал, без бэктеста. У нас порядок обратный: тики → бэктест → пейпер, с пониманием каждого куска.
- Заявленные «+20–162% в день» — ручной режим, макс. плечо, без публичных пруфов. **Не бенчмарк** для бота.
